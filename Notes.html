Beginning Concepts

World Wide Web

The World Wide Web is a componet of the Internet and is a system of servers that communicate with each other formatted documents. These formated documents are coded in Hypertext Markup Language ("HTML").

There are many types of files on the web, including: plain text, HTML, images, videos and music.

The major pieces are you, your computer running a web browser, the Internet, and servers. Your browser makes requests via the Internet to servers. These requests are using a protocol called HTTP. And the servers respond with files that the browser displays.

Basic HTML Elements

Unlike people using natural langauge, computers can't make inferences and so they interpret language literally. Computer programming languages are designed to limit ambiguity.

The basic form for HTML Markup is: <[NAME]> Contents </[NAME]>.

HTML Attributes

An HTML attribute is a modifier of an HTML element. An attribute either modifies the default functionality of an element type or provides functionality to certain element types. Attributes always have values. A common attribute is a web address that becomes a hypertext link.

The basic form to create a weblink using the anchor tag is:

<[a] href="[www.WEBADDRESS.com]"> DISPLAY TEXT </a>.

HTML images appear inline with text by using a tag with serveral attributes.

More HTML Elements

Elements can be inline or block. Inline elements occurn withing the line of the text. A block element will draw an invisible block around the text and create whitespace around it. The block is configurable with attributes. Here is a video explaning the difference.

Text can be broken up by using SPAN and DIV. Span is inline and DIV is block. Using SPAN allows inline assignment of attributes to text. Using DIV will create a block, by starting a new line and adding white space before and after the new line/text.

Tips for Coding

It's important to write code that is neat and formatted.

1. Use tabs instead of spaces.
2. Comment code when writing it.
3. Use whitespace to make code readable.
4. Before taking a break from a project, create a detailed comment of where you left off.
